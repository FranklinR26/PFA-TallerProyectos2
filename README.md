## 🚀 Problemática y Justificación del PMV
Las universidades con currículo flexible enfrentan una explosión combinatoria al intentar cuadrar horarios. Este proyecto desarrolla un **Producto Mínimo Viable (PMV)** que automatiza la asignación mediante un enfoque de **Problema de Satisfacción de Restricciones (CSP)**.

**Justificación del Stack:**
- **MongoDB:** Flexibilidad para manejar planes de estudio que cambian frecuentemente.
- **Express & Node.js:** Manejo eficiente de peticiones asíncronas durante el proceso de cálculo intensivo.
- **React:** Interfaz dinámica para la visualización de horarios sin recargas de página.

## 🛠️ Arquitectura
El sistema sigue un modelo **SPA (Single Page Application)** que consume una **API REST**. La lógica de optimización está aislada en un servicio del backend para permitir escalabilidad horizontal.
